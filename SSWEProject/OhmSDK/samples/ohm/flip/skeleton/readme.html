About Skeleton

This folder contains the skeleton sample code which demo a typical Flip application.

Building on MacOS with Apple XCode

The folder contains a project file called skeleton.xcodeproj . This file is made for XCode 3.1, but XCode 3.2 and XCode 4 are also supported.

Those targets will be described later in this readme file.

Building for XCode 3

The project should build right out of the box. By default the Base SDK is selected. Just select the appropriate target and build it.

Building for XCode 4

Before building, you should explicitely select the Base SDK by selecting it from the project build settings pane. Then, the project should build right out of the box. Just select the appropriate target and build it.

Building on Windows with Microsoft Visual Studio

The folder contains a solution file called skeleton.sln . This file is made for Visual Studio 2005 (VC 8.0), but later versions are also supported.

Note:SkeletonServer is only available on MacOS X and Linux.

Those projects will be described later in this readme file.

Important:DirectX 9c SDK is needed to build the sample projects.

If you are using Visual Studio 2005, you must download and install the February 2010 release :

http://www.microsoft.com/en-us/download/details.aspx?id=10084

If you are using a later version of Visual Studio, you shoud download and install the June 2010 release :

http://www.microsoft.com/en-us/download/details.aspx?id=6812

Building on Ubuntu 10.04 LTS

The skeleton includes a build.sh script to build the server. The script uses the stock ubuntu 10.04 gcc to compile, which is version 4.4.3 .

Building for Intel x86

By default, build.sh is made to compile an Intel x86 executable, no further configuration is needed.

Building for AMD64/EM64T

Before building, edit build.sh to change the default configuration, so that OhmstudiodArch is Linux64 

OhmstudiodArch="Linux64"
#OhmstudiodArch="Linux"
#OhmstudiodArch="MacOSX"

Building

To build, change your current directory to the skeleton folder using cd then type :

./build.sh

Or explicitly if not using bash :

bash build.sh

The resulting executable will be written in the skeleton folder.

SkeletonLocal

This target (XCode) or project (Visual) embeds a Flip server and client into a single application. It is commonly used to develop a Flip application, as this makes building, running and debugging a quick and simple task. It may also be used to develop non-collaborative application, while still retaining all Flip features that will ease your application development.

Once built with the instructions described above, just run the executable, and the application will run without any needs for more configuration.

The resulting application is described below.

SkeletonServer

This target (XCode) or build script (Linux) will build a server only executable.

WARNING:On MacOS X, you must run the server from the terminal, not from XCode.

When launching the executable with no argument, it will start the server and will wait for Flip client connection on port 9222 and will wait for http client connection on port 9223 to monitor the server. flip server’s Documents will be written in the current working directory.

By default, the server will not run as a daemon, which means that it can be closed by simply typing ctrl-C on the terminal.

The list below describes some of the available arguments when lauching the server :

--port

flip client connection port

--service_port

http client connection port

--daemon

run in daemon mode

--angel

run in angel mode

Running in daemon mode will detach the process from the terminal. Running in angel mode will automatically relaunch the server in the event of a server crash.

For example :

./SkeletonServer --port 9999 --service_port 8080 --daemon --angel

Will accept Flip client on port 9999, will run the http server on port 8080, will run as a daemon and will be protected against unexcepected server crash.

When running the executable, the output will be typically as below, until a client connects.

macpro:Debug rdi$ ./SkeletonServer
Central: Starting
Central: Setting up flip listen port
Listening on port 9222
Central: Setting up http listen port
Listening on port 9223
Central: Ready

Logs are also available on the http server. To do so, simply run your internet browser and visit the url :

http://localhost:9223/status

Server monitoring is described below.

SkeletonClient

This target (XCode) or project (Visual) will build a client only executable. This is typically the kind of application you would deliver to your customer if you are using Flip collaborative features.

Once built with the instructions described above, you should first run the server from a terminal. Then, If you are running the server on the same computer than the client, no more configuration is needed as the default host will be localhost .

If you want to connect from another computer, you can do so by using the -host argument.

When connecting with more than one client to the same session, usernames must be different and can be changed with the -username argument.

The server handles multiple sessions at once. To connect to a particular session, use the -session argument.

The list below describes some of the available arguments when lauching the client :

-host

host to connect (default: localhost )

-port

port to connect (default: 9222 )

-session

session to join (default: mysession )

-username

username to join (default: me )

Important:note that the client arguments’ syntax is different from the server

For example giving arguments :

-host=myserver.local -port=9999

Will connect on host myserver.local on port 9999 .

Using the Application

Running the application for the first time should show the following graphical user interface.

Taxis and Pedestrians can be selected individually, or with a marquee. Clicking and dragging them will move them on the screen, in a “ghosting” fashion. Taxis are constrained to the horizontal axis, and can be resized by clicking on the right end of the taxi image.

Further actions are available through the application menus.

Using the Server Monitoring Application

Once the server is running, you may access it using your internet browser by visiting the following url :

http://localhost:9223/status

Which should typically show the following :

Tabs are available at the start of the pages where :

  • Status will show the current configuration and realtime metrics of the running server
  • Logs will show last output server logs
  • Sessions will show the currently connected flip clients
  • Restart will allow to shutdown the server

Problems ?
